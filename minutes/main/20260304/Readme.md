Sebastian: I've shortened the mission statement in keeping with other recent charters.  Given our 10 year history, we can keep the charter short and link to pages with the background, e.g. the WoT documentation page. https://www.w3.org/PM/Groups/charter-assistant.html

Dave: using recent charters is good for inspiration. We can also check if we're missing anything from the template.

Sebastian: the template is helpful but we still need to develop the content ourselves.

He reviews the scope/out-of scope sections and then talks us through section 3 (deliverables). The main focus on the charter should be on TD 2.0. We also have registries and a suite of informative specifications. These include scripting, use cases, primer & best practice, and a long list of protocol bindings.

We will maintain the use cases and requirements, and work on the primer to describe best practice for web developers including a combination of AI and tooling.

The protocol bindings are published as W3C Notes. We will add new ones and maintain existing ones.

Other deliverables include test suite and implementation report, tolling and an updated set of semantic artefacts for TD 2.0.

Sebastian notes that this is a long list of deliverables, but not so much in comparison to the Verifiable Credentials charter!
